# Bearing Rules — строгие правила ГОСТ ⇄ ISO для бота Эверест

## 1. Декомпозиция обозначения

Любое обозначение подшипника раскладывается на:
```
[BRAND]  [PREFIX]  BASE  [SUFFIX_GEOMETRY]  [SUFFIX_EXECUTION]  [SUFFIX_CLEARANCE]  [SUFFIX_CLASS]
  SKF              6205          -            2RS                  /C3                 P6
```

### База
- ISO: 4–7 цифр (6205, 22208, 30206, 1726205, 7208).
- ГОСТ: 7–8 цифр (180205, 3180118).

### Суффиксы — исполнение (уплотнение/сепаратор)
| Код      | Значение                                   |
|----------|--------------------------------------------|
| `2RS`    | две резиновые уплотнения (контактные)      |
| `2RSR`   | SKF-вариант 2RS                            |
| `RS`     | одно уплотнение                            |
| `ZZ/2Z`  | две металлические шайбы                    |
| `Z`      | одна металлическая шайба                   |
| `K`      | конический внутренний диаметр              |
| `N/NR`   | канавка/стопорное кольцо на наруж. кольце  |
| `M`      | латунный сепаратор                         |
| `TN`     | полиамидный сепаратор                      |
| `E`      | усиленная конструкция                      |

### Суффиксы — радиальный зазор
`C0` (норма, опускается) · `C1` · `C2` (уменьшенный) · `C3` (увеличенный) · `C4` · `C5`

### Суффиксы — класс точности
`P0` (норма, опускается) · `P6` · `P5` · `P4` · `P2`

## 2. Правило соответствия ГОСТ ⇄ ISO

Аналог валиден **только** при одновременном совпадении:
1. **Тип** подшипника (шариковый радиальный / роликовый / упорный / сферический / игольчатый / конический).
2. **Серия размеров** (лёгкая 100, средняя 200, тяжёлая 300 и т.д.).
3. **Геометрия**: d (внутр), D (наруж), B или H (ширина/высота) — допуск = 0 мм.
4. **Исполнение**: уплотнение, сепаратор, зазор, класс — должны совпадать или быть совместимыми.

Если хотя бы один параметр не совпал → **`NO DIRECT EQUIV`**.

## 3. Распространённые соответствия (верифицированные)

**Важно:** этот список — только для ориентира, в боте ВСЕГДА проверяй через `catalog.d/D/H` перед выдачей.

| ISO      | ГОСТ     | Тип                       |
|----------|----------|---------------------------|
| 6000     | 100      | шариковый радиальный      |
| 6200     | 200      | шариковый радиальный      |
| 6300     | 300      | шариковый радиальный      |
| 6400     | 400      | шариковый радиальный      |
| 22200    | 3520     | сферический двухрядный    |
| 30200    | 7000     | роликовый конический      |
| 32200    | 7500     | роликовый конический (ш.) |
| N200     | 2200     | роликовый радиальный      |
| NU200    | 32200    | роликовый радиальный NU   |
| 7200     | 36200    | радиально-упорный ш.      |

**NEQ/MOD оговорка:** даже если ГОСТ ссылается на ISO как «не эквивалентный» (NEQ) или «модифицированный» (MOD) — это относится к стандарту как документу, **не** к конкретному подшипнику. Для продуктовой эквивалентности — только геометрия + исполнение.

## 4. Типичные ошибки бота (чего избегать)

- ❌ «6205 = 180205» без проверки d/D/B.
- ❌ «Аналог 6205-2RS → 180205» (без суффикса 2RS у ГОСТ → `NO DIRECT EQUIV`).
- ❌ Подсказать «похожий» размер (6204 вместо 6205).
- ❌ Слепо копировать кросс-референс-таблицы с вторичных сайтов.
- ❌ Утверждать, что SKF = FAG при разных суффиксах зазора.

## 5. Формат ответа по подшипнику (эталон)

```
Подшипник: 6205-2RS/C3 (SKF)
— Тип: шариковый радиальный однорядный
— Параметры: d=25, D=52, B=15 мм, масса=0.13 кг
— Исполнение: 2RS (два резиновых уплотнения), зазор C3 (увеличенный)
— Аналоги:
    • ГОСТ: 180205АС9 [[TBD]] — проверить в каталоге Эверест
    • ISO: 6205-2RS/C3 (FAG, NSK, NTN — совпадение геометрии)
— Заказ: info@ewerest.ru | наб. Пречистенская 72, Вологда
```

Если геометрия неполная:
```
Подшипник: <raw>
— В каталоге Эверест не найден. [[TBD]] — уточнить у менеджера.
— Для подбора аналога нужны: d, D, B, исполнение.
```

## 6. Реализация в коде

Модуль `src/bearings.js`:
- `detectBearing(text)` — парсит ISO/ГОСТ через регулярки.
- `normalize(raw)` — разделяет brand / base / suffixes.
- `enrichBearingContext(env, hint)` — ищет в D1 `catalog`.
- `findAnalogs(env, item)` — строгое совпадение `d = ? AND D = ? AND H = ?` + execution.

Модуль `src/prompt.js` вшивает найденный подшипник в блок `=== CATALOG HIT ===` в system prompt.

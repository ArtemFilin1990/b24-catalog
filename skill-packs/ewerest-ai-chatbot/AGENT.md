# AGENT — ewerest-ai-chatbot

## Routing

Определи режим по запросу пользователя:

| Триггер в сообщении                                              | Режим |
|------------------------------------------------------------------|-------|
| «создай бота», «с нуля», «новый worker», «инициализируй»         | A     |
| «добавь память», «добавь RAG», «подключи Vectorize», «обнови»    | B     |
| «плохо отвечает по подшипникам», «не находит аналог», «промпт»   | C     |
| «залей каталог», «загрузи документы», «ingest», «seed»           | D     |
| «аудит», «проверь бота», «что не так», «ревью»                   | E     |

## Общий чек-лист перед любым режимом

1. Прочитай `wrangler.toml` пользователя (если есть) — не переписывай bindings вслепую.
2. Прочитай `src/index.js` (если есть) — не теряй существующую логику.
3. Сверь account_id: должен быть `84cbacc4816c29c294101ec57a0bea5d`.
4. D1 database_id: `11a157a7-c3e0-4b6b-aa24-3026992db298`.
5. R2 bucket: `vedro`.

## Вывод всегда в формате

```
РЕШЕНИЕ: <одна строка>

Шаги:
1. ...

Эффект: ...

Риски:
- если X, то Y (P0/P1/P2)

Альтернативы: ...

Неизвестно: [[TBD]] + где проверить
```

## Код

- Production only, без псевдокода.
- Один блок на файл.
- Комментарии на русском.
- Импорты явные.
- Ошибки логируются + возвращаются как JSON с `error` и `code`.

## Запреты

- Не выдумывать API Cloudflare — проверить `https://developers.cloudflare.com/workers-ai/`.
- Не трогать Vectorize index name без подтверждения (может быть завязан на существующие данные).
- Не хардкодить токены — только `env.ADMIN_TOKEN`, `env.BITRIX24_WEBHOOK`.
- Не менять D1 schema без миграции (добавлять через `ALTER TABLE`).
